# PPO Pipeline (`ppo_pipeline.py`)

A concrete toy implementation of the PPO (RLHF) training pipeline using text
prompts and responses. All four models are simple linear networks so the focus
stays on the data flow and algorithm, not the architecture.

## The Four Models

| Model | Role | Trained? | Input | Output |
|---|---|---|---|---|
| `PolicyModel` | Actor — generates the next token | Yes | state embedding `[B, embed_dim]` | logits `[B, vocab_size]` |
| `ReferenceModel` | Frozen copy of the initial policy | No (frozen) | state embedding `[B, embed_dim]` | logits `[B, vocab_size]` |
| `RewardModel` | Scores the quality of a completed response | No (frozen) | full sequence tokens `[B, prompt_len+T]` | scalar reward `[B]` |
| `ValueModel` | Critic — estimates expected future reward | Yes | state embedding `[B, embed_dim]` | scalar value `[B]` |

## Tensors and Shapes

| Tensor | Shape | Meaning |
|---|---|---|
| `prompts` | `[B, prompt_len]` | Integer token IDs of the input prompt |
| `actions` (response) | `[B, T]` | Integer token IDs generated by the policy |
| `states` | `[B, T, embed_dim]` | Mean-pool embedding of `(prompt + response[:t])` at each step |
| `rewards` | `[B, T]` | Sparse: `0` for all steps except the last, where `RewardModel` is called |
| `values` | `[B, T]` | `ValueModel(state_t)` — baseline for advantage computation |
| `logp_old` | `[B, T]` | Log-prob of actions under the policy **at rollout time** (frozen) |
| `logp_new` | `[B, T]` | Log-prob of actions under the policy **at update time** (has grad) |
| `logp_ref` | `[B, T]` | Log-prob of actions under the frozen reference policy |
| `advantages` | `[B, T]` | How much better each action was than the value baseline |
| `returns` | `[B, T]` | Advantage + value — target for the value loss |

## Training Loop

```
for each iteration:
    1. collect_rollout()        autoregressive token generation
         for t in range(T):
             context  = prompt + response[:t]
             state    = TokenEmbedder(context)      mean-pool embedding
             action   = policy.sample(state)        next token
             logp_old = policy.log_prob(state, action)
             logp_ref = ref_policy.log_prob(state, action)
             value    = value_model(state)
         rewards[:, -1] = reward_model(prompt + full_response)   sparse reward

    2. compute_advantages_gae()
         δ_t = r_t + γ·V(s_{t+1}) − V(s_t)
         A_t = Σ (γλ)^k · δ_{t+k}
         R_t = A_t + V(s_t)

    3. ppo_update()
         logp_new   = policy.log_prob(states, actions)     recompute w/ grad
         values_new = value_model(states)

         policy backward:  policy_loss + kl_coef * kl_loss
         value  backward:  value_loss
```

## Advantage Estimation

Two implementations are provided; swap one line in `collect_rollout` to switch:

### Monte Carlo (`compute_advantages_mc`)

\[
R_t = \sum_{k \ge 0} \gamma^k r_{t+k}, \quad A_t = R_t - V(s_t)
\]

Uses the full observed return. High variance, zero bias.

### GAE (`compute_advantages_gae`)

\[
\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t), \quad A_t = \sum_{k \ge 0} (\gamma\lambda)^k \delta_{t+k}
\]

Uses TD residuals with exponential decay. Lower variance, small bias from \(V\).

| `λ` | Equivalent to | Variance | Bias |
|---|---|---|---|
| `1.0` | Monte Carlo | High | Zero |
| `0.0` | One-step TD | Low | High |
| `0.95` | Typical PPO | Balanced | Balanced |

With sparse reward (only at the last token), GAE is particularly helpful: it
uses \(V(s_t)\) to propagate the terminal reward signal back across all token
positions smoothly.

## PPO Loss Terms

```
policy_loss  = -E[ min( r_t · A_t,  clip(r_t, 1-ε, 1+ε) · A_t ) ]
               where  r_t = exp(logp_new - logp_old)

value_loss   = E[ (returns - values_new)² ]

kl_loss      = E[ logp_new - logp_ref ]   (stay near reference policy)

total_loss   = policy_loss + vf_coef * value_loss + kl_coef * kl_loss
```

Policy and value models are updated with **separate backward passes**:

```python
# policy head only
(policy_loss + kl_coef * kl_loss).backward(retain_graph=True)
policy_optimizer.step()

# value head only
value_loss.backward()
value_optimizer.step()
```

## LLM / RLHF Analogy

| This code | Real RLHF |
|---|---|
| `VOCAB` (16 words) | Full tokenizer vocabulary (50k+ tokens) |
| `TokenEmbedder` (mean pool) | Transformer with causal attention |
| `state_t` | Hidden state at position `t` in the transformer |
| `action_t` | Token sampled at position `t` |
| `RewardModel` (scored once) | Human preference model, scored on the full response |
| `ref_policy` | Frozen SFT (supervised fine-tuned) checkpoint |
